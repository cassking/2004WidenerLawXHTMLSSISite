Perched Upon a Lily Pad

#leftbox {
  position: absolute;
  left: 0px;
  width: 215px;
  }

This fixed width sidebar, #leftbox, is absolutely positioned with a left position given but no top positioning. This is so it will automatically position itself based on the size of the #header div.

This box gets its background color from the background color set in the <body> tag. This way it will appear to run the full height of the page.

#header {
background-image:
  url("images/waterlilies6.jpg");
background-position: 0 0;
background-repeat: no-repeat;
height: 100px;
}

The #header div at the top of the page spans the width of the viewport. It has a background image applied and is given a height so that the following absolutely positioned divs are placed properly on initial load of the page.

#rightbox {
  position: absolute;
  right: 0;
  margin-right: 0;
  width: 138px;
  }

This absolutely positioned #rightbox has a fixed width and right positioning set. Again, no top positioning is given so this div will adjust its height to what comes before it.

The background-color on this div is made by a repeating .gif placed on the #container div.

Because this div, and the #leftbox div are both absolutely positioned and have fixed widths, care must be taken with content that may not wrap, such as long URIs or <pre> tags. IE will expand the divs to contain the content, which violates the specs, and other browsers will let too wide content spill outside of the div.

IE5/Mac presents an additional challenge with this right positioned AP div. See source for more information and the hack used to overcome the problem.

Return to p.i.e.

Following #leftbox, a #container div is opened which will hold the rest of the page. This div is given a left margin equal to the width of the absolutely positioned (AP) div preceding it. In this case, #container has a left border and a repeating background image on the right that provides the background color for the #rightbox column.

#container {
  margin-left: 215px;
  background-image: url("rightbg.gif");
  background-position: 100% 0;
  background-repeat: repeat-y;
  }

Inside #container, a static div, #navmenu, holds nav links. The links are marked up as a horizontal unordered list. 

#navmenu ul {
  display: block;
  list-style: none;
  text-align: center;
  }
#navmenu ul li {
  display: block;
  float: left;
  width: 10em;
  }

Inside the <ul>, the <li>s are floated left and given a width and both the <ul> and the <li> are given a display property of {display: block}. Following the floats is a <span> that clears them. This menu will wrap nicely if the browser window is made narrow enough or the text is made large enough to cause that.

#wrapper {
  margin-right: 138px; 
  }

A #wrapper div is opened to hold a couple of floats. Since it is within the #container, it needs only a right margin to clear the AP #rightbox.

IE5.5 will need a width on this #wrapper to prevent a horizontal scrollbar, but IE6 must not have a width. See source for the hack.

.left {
  float: left;
  width: 43%;
  }
.right {
  float: right;
  width: 43%;
  }

Inside the #wrapper are two float containers, one floated left and one floated right. Nothing extraordinary about either of these.

Since these divs are both floated, there will be no 3px text-jog in IE if one div is longer than the other.

IE doubles the margins of the floats on the side they contact their container, #wrapper in this case, and so the correct margin width for each float must be hacked for other browsers. See source.

.clear {
  clear: both;
  margin: 0;
  padding: 0;
  }

Following the floats is a clearing div with {clear: both}.

IE5/Mac needs the margin and padding of this div set to 0 (zero). Because of this, if it was desired to put text in this clearing div, or put a border on it, it would need to extend the full width of the #wrapper. In this demo, it just holds a space.

 
#contents {
  margin-right: 149px;
  margin-left: 10px;
  }

Another div contains the rest of the content of the page. This div, #contents, needs a right margin to clear the right AP div. Further margin width can be given to left and right if the background of #container is desired as a frame for #contents (as is done here).

Note: Due to the formatting of the text of this demo, which includes <pre> tags, this particular page will look rather messy if the browser window is narrowed significantly or if very large font sizes are used. If you would like to try resizing the browser or increasing the text size, please see this example page, which is made without <pre> tags. More layout tips and known problems can also be found on the same page.
